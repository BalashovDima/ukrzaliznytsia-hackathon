# Як працює система

Система допомагає автоматизувати та оптимізувати розподіл порожніх вагонів під заявки клієнтів. Головна мета — мінімізувати сумарні витрати на порожній пробіг (відстань, яку вагон долає без вантажу від поточної станції до станції навантаження).
Щоб продемонструвати ефективність математичного підходу, система одночасно розраховує два сценарії: «Базовий» та «Оптимізований».

## Базова (жадібна, наївна, дурна) логіка

Цей алгоритм імітує роботу диспетчера, який не має інструментів візуалізації та автоматизації. Він працює за принципом «перший вільний — першій заявці».

- **Принцип роботи**: Система бере список заявок і для кожної з них призначає перший доступний вагон потрібного типу з загального списку парку. При цьому відстань не враховується взагалі.
- **Реальність**: Це призводить до ситуацій, коли під заявку в Одесі може бути призначений вагон, що стоїть у Харкові, хоча поруч у Миколаєві є інший вільний склад (який пізніше "піде" на іншу випадкову заявку).
- **Результат**: Випадковий розподіл, який часто створює величезні зайві витрати через хаотичне переміщення вагонів по країні.

## Розумна (оптимізована) логіка

Цей алгоритм використовує Угорський метод (алгоритм Куна-Манкреса) для розв'язання задачі про призначення.

- **Глобальний підхід**: На відміну від простого пошуку «найближчого вагона» для однієї конкретної заявки, наша система розглядає всі заявки та всі вільні вагони одночасно.
- **Математична модель**: Система будує матрицю вартостей, де в кожній клітинці — реальна відстань між вагоном та станцією навантаження через залізничний граф України. Алгоритм шукає таку комбінацію призначень, при якій загальна сума кілометрів усіх вагонів буде мінімально можливою.
- **Стратегічна вигода**: Іноді система може відправити вагон до дещо дальшої станції, якщо це дозволить іншому вагону значно скоротити свій шлях, що в сумі дасть кращий результат для всієї мережі.

## Як рахується економія?

Для кожного набору заявок система проводить «паралельний всесвіт»:

1. Рахує витрати для Оптимізованого плану.
2. Рахує витрати для Базового плану (на тих самих даних).
3. Різниця між ними — це і є реальна економія, яку ви бачите на дашборді.

### Ключові параметри:

- Вартість: Кожен кілометр порожнього пробігу оцінюється у 20 грн.
- Граф доріг: Відстані рахуються не по прямій, а через реальні залізничні сполучення між 25 вузловими станціями України.
- Типізація: Система суворо контролює відповідність вагонів типу вантажу (наприклад, зерновоз не буде призначений під щебінь).

Завдяки такому порівнянню, видно не просто цифри, а конкретну суму коштів, яку підприємство втрачало б при бездумному ручному або хаотичному управлінні парком.
